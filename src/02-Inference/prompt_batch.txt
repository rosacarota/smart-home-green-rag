Project context:
The project aims to build a RAG knowledge base for modifying existing smart-home trigger-action rules into greener versions.
A useful green smart-home rule improves sustainability by reducing unnecessary energy or resource consumption while preserving the user's functional intent, comfort, and safety. A rule can be considered green when it avoids wasteful device usage, adapts actions to occupancy or presence, uses environmental context such as temperature, humidity, daylight, rain, or weather, optimizes timing or duration of energy-intensive actions, reduces standby consumption, or supports monitoring of inefficient behavior.
Green relevance should be assigned only when the rule can reasonably contribute to energy saving, water saving, HVAC optimization, lighting efficiency, appliance scheduling, or resource-aware automation, not merely because it involves a smart-home device.

Rule-level green relevance labels:
- high: the rule either clearly reduces/optimizes energy or resource use, or is highly useful for generating a greener variant for a high-impact smart-home system such as HVAC, thermostat, irrigation, lighting, appliance, plug, or standby control.
- medium: the rule has plausible green value, but is partly comfort, convenience, schedule, voice-control, or notification oriented.
- low: the rule is not directly green, but can be used as a weak baseline for a greener variant.
- none: the rule is unrelated, ambiguous, decorative, contradictory, or not useful for green rule generation.
